# デモ動画 台本（5分）

Grow の価値 **「使うほど、人とAIが共に育つ」** を、**一筆書き**で見せる5分構成。本番環境（<https://grow-c6m2ic6jcq-an.a.run.app>）＝実 Gemini で撮影する。総尺の目安は約 300 秒。

伝えたい一貫メッセージは、**「カードに積み重なる履歴の上で人とAIが交互に手を入れ、その履歴からAIは働き方を学んで成長し、人も任せ方を磨いていく」** こと。

**通しの流れ:** カード作成 → 受付AIが自分から質問 → オートノミー L2 設定 → オートパイロット → 指揮者が判断 → 成果物のライブ・ストリーミング生成 → レビューAIが差し戻し → 再生成 → 人が承認 → ✧ 学ぶ → 再生成の diff で "共に育つ" を証明 → 意思決定トレースと実コストで締め。

> UI ラベルは実装準拠: 「AIにまかせる」「オートパイロット」「AIと壁打ち / 分解」「差し戻す / 理由を付けて差し戻す」「完了にする」「✧ 学ぶ」、ドロワー上部の「オートノミー・ダイヤル（L0〜L3）」＋「Web検索トグル / コスト上限」、右上「◈ ナレッジ」、ドロワー内「◈ 適用ルール」「意思決定トレース」。

---

## シーン0 — フック（0:00〜0:20 / 20秒）

- **画面:** 看板ボード全体。カードが5レーンに並び、進行中レーンで AI稼働カードのドット（ティール）が点滅。
- **操作:** なし（ボードを軽くパン）。
- **ナレーション:** 「AIを仕事に取り入れると、ツールの往復・毎回の説明・勝手な暴走が面倒です。Grow は、人とAIが同じカードの上で協働する場。やり取りの履歴が積み重なるほど、AIはあなたの働き方を覚え、あなたもAIの任せ方が上手くなっていきます。」
- **映すべきポイント:** 上部カウンタ「あなたの番 N ／ AI稼働 N」、カード左端の色バー（AI=ティール／人=アンバー）。**積み重なる履歴の上で「次に手を入れるのは誰か」**が一目で分かることを最初に印象づける。

---

## シーン1 — 積み重なる履歴を見せる（0:20〜0:45 / 25秒）

- **画面:** 既存カードを1枚クリックしてドロワーを開く。
- **操作:** T-098「競合調査レポートの下書き」など AI作業中カードを開き、アクティビティ・スレッド（人とAIのコメントが交互に積み重なっている）と役割バッジ（計画AI／実行AI…）を見せる。
- **ナレーション:** 「カードには人とAIのやり取りが全部残ります。誰のコメントか、どの役割のAIかがバッジで分かる。この積み重なった履歴が、あとでAIの学びの源になり、資産になります。」
- **映すべきポイント:** コメントの役割バッジ（#19）、ステータスバッジ、ドロワー上部のオートノミー・ダイヤル。履歴が消えずに残っていること。

---

## シーン2 — カード作成 → 受付AIが自分から質問（0:45〜1:20 / 35秒）

- **画面:** ToDo レーン末尾の「＋ カードを追加」。
- **操作:** 新規カードを作成（例: タイトル「主要な生成AIコーディングツールを調査」、ラベル `調査`）。作成直後にドロワーが開く。数秒待つと**受付エージェント（#27）が自分から確認質問を投稿**する。
- **ナレーション:** 「新しいタスクを放り込むと、受付AIが『実行できるか／もっと聞くべきか／分解すべきか』を自分で判断します。今回は前提が足りないと見て、AIの側から質問してきました。これも履歴の一行になります。」
- **映すべきポイント:** 計画AIバッジ付きの質問コメントが**人の操作なしに**現れること。「AIが役割の範囲で自律的に動く」ことの最初の証拠。
- **注意:** 受付AIの応答は Flash で数秒。待ち時間は編集でトリムして良い。

---

## シーン3 — オートノミーを L2 に設定（1:20〜1:40 / 20秒）

- **画面:** ドロワー上部のオートノミー・ダイヤル。
- **操作:** 受付AIの質問にコンポーザで一言回答（例「対象は5社、料金と主要機能を表で」）。続いてダイヤルを **L2「承認後は自動」** に切り替え、Web検索トグルが ON、コスト上限が設定できることを一瞬見せる。
- **ナレーション:** 「AIにどこまで任せるかはカード単位で決められます。L2は『承認後は完了まで自動』。Web検索の可否やコスト上限も、ここで縛れます。こうやって"任せ方"を調整するのも、使ううちに上手くなっていきます。」
- **映すべきポイント:** L0〜L3 ダイヤルと行動範囲ポリシー（#21）。「勝手に進めすぎない」を人side が制御で担保していること。

---

## シーン4 — オートパイロット → 指揮者が判断 → ライブ生成（1:40〜2:40 / 60秒）

- **画面:** ActionBar の「オートパイロット」ボタン。
- **操作:** 「オートパイロット」を押す。指揮者AI（#22）が「引き受けました」→「次のアクションを『実行』と判断しました（理由: …）」を投稿し、実行AIへ処理を引き継ぐ。**成果物がストリーミングで少しずつ画面に書き出されていく**（#24）。進捗コメントも SSE で増える。
- **ナレーション:** 「オートパイロットを押すと、指揮者AIが『次に何をすべきか』を自分で判断し、実行AIへ引き継ぎます。ここがポイント。着手前に、このカードに関係する“働き方のルール”を履歴から読み込んでいます。」
- **映すべきポイント:**
  - 指揮者AIの**判断理由コメント**（なぜその一手か）。
  - ドロワーの「◈ 適用ルール」に、retrieval されたルール（K-01／K-03 等）が**着手前に可視化**されること。AIが何を学んで前提にしているかがブラックボックスにならない。
  - 成果物が**リアルタイムに生成**される様子（ストリーミング）。実 Gemini + Google Search grounding で動いている実感。
- **注意:** execute は `gemini-2.5-pro` ＋ grounding で 20〜40秒かかりうる。生成の“流れる”瞬間は見せ場なので冒頭は等速、中盤を軽く早送りして良い。

---

## シーン5 — レビューAIが差し戻し → 再生成（2:40〜3:20 / 40秒）

- **画面:** アクティビティ・スレッド。
- **操作:** 成果物保存後、**レビューAI（#23）が自己採点**して「修正が必要と判断しました。実行AIに差し戻します」＋【レビュー指摘】を投稿 → 実行AIが再生成（最大2周）。
- **ナレーション:** 「面白いのはここ。AIがAIの成果物をレビューして、基準を満たさなければ自分で突き返します。人が指摘しなくても、品質のループがAIの中で回る。役割の違うAIが履歴の上で協働しているんです。」
- **映すべきポイント:** レビューAIバッジ、【レビュー指摘】コメント、差し戻し→再生成でバージョンが増えること。「多エージェントが自律協働している」核心。

---

## シーン6 — 人の承認ゲート（3:20〜3:35 / 15秒）

- **画面:** ステータスが「あなたのレビュー待ち（you_review）」になり、レビューレーンへ移動。
- **操作:** 内容をざっと見て「完了にする」を押す（＝人の最終承認）。必要なら「差し戻す」で理由を付けて返せることも一言添える。
- **ナレーション:** 「完成してもAIは勝手に終わらせません。次に手を入れる番が人に移り、承認するのも、理由を付けて差し戻すのも人。ここが暴走を防ぐ最後のゲートです。」
- **映すべきポイント:** 人の承認ゲート＝次の番が人に戻ること。制御と自律の両立。

---

## シーン7 — ✧ 学ぶ（3:35〜4:05 / 30秒）

- **画面:** 完了カードのドロワーに現れる「このやり取りから働き方のルールを学べます」＋「✧ 学ぶ」。
- **操作:** 「✧ 学ぶ」を押す → ナレッジ抽出AI（#13）がルール候補を提示（例「調査レポートは料金を表形式にし各項目に出典URLを付ける」scope=個人）。1件を**採用**。
- **ナレーション:** 「このやり取りの履歴から“働き方のルール”を蒸留します。採用するかは人が決める。間違った学びが勝手に溜まらないように。ここでAIが一歩育ちます。」
- **映すべきポイント:** ルール候補（scope／確度／出典付き）、採用でナレッジに NEW が付くこと。「履歴が資産に変わり、AIが学ぶ瞬間」。

---

## シーン8 — 共に育つ証明：再生成の diff（4:05〜4:40 / 35秒）

- **画面:** 同じ `調査` タグの別カード（例 T-104「競合SaaS 5社の料金プランを調査」）で「AIにまかせる」を実行 → 新バージョン生成後、「意思決定トレース／差分リプレイ」を開く。
- **操作:** 差分リプレイ（#20）で **前バージョン → 新バージョン** を再生。ルール適用フラッシュで、いま採用したルールが効いた箇所がハイライトされる。
- **ナレーション:** 「別の調査タスクで同じように任せると、さっき学んだルールが2回目の成果物に反映されています。差分で見ると一目瞭然。指示を細かく書き直していないのに、AIが自分でフィットしにきている。これが“使うほど共に育つ”ということです。」
- **映すべきポイント:** diff で before→after の違い、適用ルールに**新しく学んだルールが増えている**こと。ナレッジ・ループが閉じた証拠。

---

## シーン9 — 意思決定トレースと実コストで締め（4:40〜5:00 / 20秒）

- **画面:** ドロワーの「意思決定トレース」。
- **操作:** 成果物の版ごとに「どのジョブが・どのルール（K-xx）を前提に・何トークン／$いくらで生成したか」を見せる。右上「◈ ナレッジ」を一瞬開き、確度・適用回数・出典と夜間メンテナンスの受信箱にも触れる。
- **ナレーション:** 「なぜAIがこう動いたかは全部説明できます。前提にしたルール、使ったトークン、実コスト。この競合調査は実際に約 $0.02。ブラックボックスにしない。使うほど人とAIが共に育つ——それが Grow です。」
- **映すべきポイント:** 説明可能性（トレース #25）、実コスト算定、ナレッジの健全性（夜間CI #26）。締めのメッセージ。

## 撮影前チェックリスト

**デモシード状態:**
- [ ] `infra/20_migrate.sh --seed` 直後の状態、または `make db-reset && make migrate && seed` で**シードデータ（T-104 競合調査ほか、T-098、K-01〜K-05）**が入っていること。
- [ ] シーン8の「共に育つ証明」を確実に見せるため、学ばせるルールと再実行するカードのラベル（`調査`）が retrieval で一致することを事前確認（T-104 は `調査` タグなので K-01／K-02／K-03／K-04 が retrieval される）。
- [ ] 「✧ 学ぶ」で採用するルールは事前に候補内容を確認し、狙った1件を採用できるようリハーサルしておく。

**実 Gemini のレイテンシ対策:**
- [ ] 撮影直前に Cloud Run を**プリウォーム**（一度ボードを開いて scale-to-zero から起こしておく）。冒頭の 200 OK 待ちを撮影に含めない。
- [ ] execute（`gemini-2.5-pro` ＋ grounding）は 20〜40秒かかりうる。ストリーミングの“流れ出す”瞬間は等速で見せ、長い生成中盤は編集で軽く早送りする方針を決めておく。
- [ ] 受付AI・指揮者・レビューは Flash で数秒。待ちはトリムして良いが、「人が操作していないのにAIが動く」ことが伝わる長さは残す。
- [ ] 差し戻しループ（#23）が起きない可能性もあるため、起きなかった場合の代替（シーン6の「差し戻す」を人が実演）も用意。

**環境:**
- [ ] Cloud SQL の activation-policy が `ALWAYS`（停止していると 500 になる）。
- [ ] `AI_PROVIDER=gemini` で稼働（本番）。ローカル Mock では出典・実コストが疑似値になるので**本番で撮る**。
- [ ] 想定外コストを避けるため、コスト上限ポリシーを一部カードに設定しておくと安全。
- [ ] ブラウザは通知・拡張を切り、ウィンドウ幅はデスクトップ想定（MVP はデスクトップ専用）。
