# 04 · デザイントークン

採用デザインは **案B「Signal」**（情報密度高め・プロツール感・エージェントの状態が一目で追える）。プロトタイプ `Grow.dc.html` から抽出した確定値。実装ではこれらを CSS変数 / テーマトークンにして使う。

## 4.4 アニメーション

| 名前 | 定義 | 用途 |
|---|---|---|
| `batonpulse` | 1.4s（attentionは1.6s） infinite; opacity/ scale が `1→.3/.6→1` | 稼働中・要対応のステータスドットを脈打たせる |
| `batonin` | .22s ease; `translateX(16px)+opacity0 → 0` | ドロワーのスライドイン |
| `msgin` | .2s ease; `translateY(6px)+opacity0 → 0` | コメント/チャット/提案/オーバーレイの出現 |
| card hover | transition .12–.15s; box-shadow ＋ translateY(-1px) | カードのホバー |

**AI実作業のタイミング（プロトタイプの擬似値・本実装では実イベントに置換）:**
- 着手 → 進捗 0%
- +1600ms → 進捗 45% ＋ 中間コメント
- +4200ms → 進捗 100% ＋ 完了コメント ＋ レビューへ
- 壁打ち: 人の送信 → +850ms でAI応答＋分解候補提示

## 4.5 アイコン対応（プレースホルダ → 実装）

プロトタイプは記号文字で代用。実装ではアイコンライブラリに置換推奨（Lucide 例）。

| 箇所 | プロト | 推奨アイコン |
|---|---|---|
| ロゴマーク | "B"（角丸四角） | ロゴ画像 or "B" のまま |
| ナレッジ | `◈` | `sparkles` / `brain` / `book-open` |
| 適用ルール見出し | `◈` | `sparkles` |
| 学ぶボタン | `✧` | `graduation-cap` / `lightbulb` |
| チームルール見出し | `◎` | `users` |
| 昇格 | `↑` | `arrow-up-circle` |
| 閉じる | `✕` | `x` |
| 追加 | `＋` | `plus` |
| コメント数 | `●`＋数字 | `message-circle` |
| 戻る（壁打ち） | `←` | `chevron-left` |
| ステータスドット | `●` | 塗り円のまま（点滅はCSS） |
